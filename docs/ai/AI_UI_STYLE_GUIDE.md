# AI_UI_STYLE_GUIDE.md

> 컴포넌트 / 영역별 시각 스타일 가이드.
> 본 문서는 디자인 설계 단계의 **명세** 입니다. 실제 CSS / HTML / JS 적용은 AI 기능 구현 / 검증 완료 후 진행합니다.
> 색·간격·라운드·그림자 같은 **수치값**은 [AI_DESIGN_TOKENS.md](AI_DESIGN_TOKENS.md) 에 정의됩니다.

---

## 0. 전제

- 기존 탭 / 메뉴 / 기능 / 필드 이름은 **변경하지 않음** (디자인만 변경).
- 본 가이드는 토큰을 인용해 사용 규칙을 정의합니다 (예: `card-shadow`, `radius-card`, `text-primary`).
- 토큰은 [AI_DESIGN_TOKENS.md](AI_DESIGN_TOKENS.md) 의 정의를 따릅니다.

## 1. 레이아웃 / 페이지 골격

### 1.1 페이지

- 배경: `background` (밝은 따뜻한 회색)
- 좌우 패딩: `page-padding`
- 메인 영역과 사이드바 사이 간격: `section-gap`
- 페이지 전체 폭은 너무 넓지 않게, 중앙 컨테이너에 정렬

### 1.2 헤더 / 상단바 (있는 경우)

- 높이: `header-height`
- 배경: `surface` 또는 `card`
- 하단 보더: `border-width` `border` (얇고 연하게)

### 1.3 사이드바 / 탭 영역

- 폭: `sidebar-width`
- 배경: `surface`
- 항목 간격: `card-gap` 의 절반 정도
- 선택 항목: 배경 `primary-soft`, 텍스트 `primary`, 좌측 강조 라인 또는 굵기로 강조
- 비선택 항목: 텍스트 `text-secondary`, hover 시 `text-primary` + 배경 미세 변경
- **탭 이름 / 메뉴 이름 변경 금지**

## 2. 카드

### 2.1 기본

- 배경: `card` (흰색)
- 테두리: `card-border` (`border-width` `border`)
- 라운드: `radius-card`
- 그림자: `card-shadow` (매우 약함)
- 내부 패딩: `card-padding`
- 카드 간 세로 간격: `card-gap`

### 2.2 헤더 / 본문 / 푸터

- 카드 헤더: 제목 (`card-title`) + 보조 액션 영역
- 헤더와 본문 사이 구분선은 **선택적** (없는 편이 깔끔)
- 푸터: 액션 버튼 영역 (`form-gap` 만큼 본문과 간격)

### 2.3 hover / focus

- 일반 정보 카드는 hover 효과 없음 (정적)
- 클릭 가능한 카드(예: 환자 후보)는 hover 시 `hover-shadow` 와 미세한 배경 변경

### 2.4 적용 대상

- AI 예약 도우미 / AI 휴무 도우미
- 예약 표 컨테이너
- 환자 상세 정보 / 신환 등록 후보
- 예약 후보 카드 / 휴무 후보 카드
- 통계 요약 카드
- 관리자 설정 카드
- AI 로그 상세 카드

### 2.5 주의

- 그림자를 강하게 쓰지 않음.
- 카드 안에 정보를 너무 빽빽하게 채우지 않음 (여백 확보).
- 표는 카드 안에 배치하되 가독성을 해치지 않게 한다 (§ 4 참조).

---

## 3. 사이드바 / 탭 강조

| 상태 | 스타일 |
|---|---|
| 선택됨 | 배경 `primary-soft`, 텍스트 `primary`, 좌측 또는 하단 4px 강조 (라운드 `radius-sm`) |
| 비선택 | 배경 `transparent`, 텍스트 `text-secondary` |
| hover | 배경 미세 변경, 텍스트 `text-primary` |
| 비활성 | 텍스트 `disabled`, 클릭 비활성 |

- 아이콘 + 텍스트 조합은 기존 구조 유지 (있으면 유지).
- 글자 크기·굵기 일관 — 변경 금지 항목은 그대로.

## 4. 표

이 프로그램의 핵심. 가독성이 가장 중요합니다.

### 4.1 컨테이너

- 표는 흰색 카드 안에 배치.
- 표 좌우 패딩은 카드 패딩과 동일.

### 4.2 헤더 행

- 배경: `surface` 또는 매우 연한 회색.
- 텍스트: `table-header` (굵기로 구분)
- 정렬: 텍스트 left, 숫자·시간 right, 상태·체크박스 center.

### 4.3 본문 행

- 행 높이: `table-row-height` (좁지 않게)
- 텍스트: `table-body`
- 구분선: 하단 보더만 `border-width` `border` (얇고 연하게).
- hover: 배경 미세 변경 (은은하게).
- 클릭 가능한 행: 마우스 cursor pointer.

### 4.4 셀 패딩

- 좌우 패딩: `table-cell-padding`
- 상하 패딩: `table-cell-padding` 의 절반

### 4.5 셀 안 컴포넌트

- 상태값 → 작은 badge / pill (§ 7).
- 액션 버튼 → 작은 보조 버튼, 너무 크지 않게.
- 체크박스 → 텍스트와 수직 중앙 정렬.

### 4.6 정렬 / 가독성 규칙

- 같은 열은 같은 정렬.
- 시간 / 날짜 / 숫자는 모노스페이스 검토 가능 (필요 시).
- 중요한 정보는 **색상보다 위치 / 굵기**로 우선 강조.
- 글자 크기 너무 작게 하지 않음.

### 4.7 적용 대상

- 예약관리 표 / 환자 목록 / 치료사 목록 / 의사 목록 / 휴무일 목록 / 예약문자 대상 목록 / 통계 표 / AI 로그 표.

### 4.8 주의

- 표 안 버튼이 많아도 복잡해 보이지 않게 정리.
- 기존 기능명·필드명 변경 금지.

## 5. 입력창 / 검색창

### 5.1 기본

- 배경: `card`
- 테두리: `border-width` `border`
- 라운드: `radius-input`
- 높이: `input-height`
- 좌우 패딩: 충분히 (대략 12~16px).
- placeholder: `text-muted`
- 텍스트: `text-primary`

### 5.2 상태

| 상태 | 스타일 |
|---|---|
| 기본 | `border` |
| hover | 보더 살짝 진해짐 |
| focus | `focus-ring` (포인트 컬러 외곽선) |
| 오류 | 보더 `danger`, 보조 텍스트 `danger` |
| 비활성 | 배경 연한 회색, 텍스트 `disabled` |

### 5.3 AI 명령 입력창

- 폭: 가능한 넓게 (검색창 + 한 문장 명령이 편하게 들어갈 정도).
- 우측에 `[AI 확인]` 주요 버튼.
- 하단에 예시 문구 (`text-muted`, `caption`).

## 6. 버튼

### 6.1 종류 / 용도

| 종류 | 용도 | 스타일 요지 |
|---|---|---|
| Primary | 주요 액션 ([예약 등록], [AI 확인], [휴무 등록]) | 배경 `primary`, 텍스트 흰색, hover `primary-hover` |
| Secondary | 보조 ([취소], [접기], [펼치기]) | 배경 `card` 또는 `surface`, 보더 `border`, 텍스트 `text-primary` |
| Ghost / Tertiary | 부가 ([초기화] 등) | 배경 transparent, 텍스트 `text-secondary`, hover 시 배경 미세 |
| Danger | 위험 ([삭제] 등) | 배경 `danger`, 텍스트 흰색 |
| Disabled | 비활성 (조건 미충족 시 [예약 등록]) | 배경 `disabled`, 텍스트 흰색·`text-muted`, cursor not-allowed |

### 6.2 공통 규칙

- 높이: `button-height` 통일 (입력창과 정렬).
- 라운드: `radius-button` 통일.
- 패딩: 좌우 14~18px (텍스트 길이에 따라).
- 텍스트: 짧고 명확하게 (기존 버튼명 유지).
- 아이콘 + 텍스트 조합은 8~10px 간격.

### 6.3 적용 대상

- 예약 등록 / 취소 / 수정
- AI 확인 / 접기 / 펼치기 / 초기화
- 신환 등록 / 환자 선택 / 치료항목 선택
- 관리자 설정 저장 / 위험 작업

## 7. 상태 Badge / Pill

### 7.1 기본

- 형태: 작은 pill (`radius-badge`)
- 높이: `badge-height`
- 좌우 패딩: 8~10px
- 텍스트: `badge-text` (소형, 약간 굵게)
- 배경: 상태별 `*-soft`
- 텍스트 색: 상태별 진한 톤

### 7.2 상태별 매핑 (요약)

| 상태 | 배경 | 텍스트 |
|---|---|---|
| 예약중 / 진행 | `info-soft` | `info` |
| 완료 / 등록 완료 / 성공 | `success-soft` | `success` |
| 취소 | `danger-soft` (차분) | `danger` |
| 대기 / 승인 대기 | `warning-soft` (옅게) 또는 `info-soft` | 대응 텍스트 |
| 선택 필요 (환자/치료항목/신환) | `warning-soft` (베이지 / 연 옐로우) | `warning` |
| 검증 실패 / 등록 실패 | `danger-soft` | `danger` |
| 휴무 / 오전반차 / 오후반차 | `info-soft` 변형 또는 별도 정의 | 대응 텍스트 |

### 7.3 규칙

- 같은 상태는 **모든 화면에서 동일 색상**.
- 너무 원색적이지 않게.
- 상태 글자는 한국어 그대로 (예: "환자 선택 필요").
- badge 만으로 의미가 약하면 옆에 짧은 보조 텍스트 추가.

### 7.4 적용 상태 예시

- 예약중 / 완료 / 취소 / 대기
- 선택 필요 / 승인 대기 / 검증 실패
- 등록 완료 / 등록 실패
- 휴무 / 오전반차 / 오후반차
- 신환 등록 필요 / 치료항목 선택 필요 / 환자 선택 필요

## 8. 메시지 (성공 / 경고 / 오류 / 정보)

### 8.1 종류

| 종류 | 배경 | 보더 / 텍스트 | 용도 |
|---|---|---|---|
| 성공 | `success-soft` | `success` | "예약 등록이 완료되었습니다" 등 |
| 경고 | `warning-soft` | `warning` | "기존 예약과 충돌이 있습니다" 등 |
| 오류 | `danger-soft` | `danger` | "AI API 호출에 실패했습니다 (기존 기능은 정상)" 등 |
| 정보 | `info-soft` | `info` | 안내 / 가이드 |

### 8.2 형태

- 라운드: `radius-md`
- 좌측 4px 강조 라인 (선택)
- 좌측 아이콘 + 본문 텍스트 (간격 `card-padding` 절반)
- 카드 안에 들어갈 때는 카드 본문 폭 그대로 사용

### 8.3 표현 규칙

- "예약 완료" 같은 단정 표현은 **승인·실행 후에만** 사용.
- 승인 전엔 "예약 후보입니다" / "승인하면 예약이 등록됩니다".
- 오류 메시지는 사용자에게 책임 전가하지 않게 차분하게.

---

## 9. 폼 / 라벨 / 도움말

### 9.1 폼 라벨

- 위치: 입력창 위.
- 폰트: `body` 또는 살짝 작게.
- 필수 표기: 한국어 라벨 옆 (필수) 또는 작은 별표.

### 9.2 도움말 / 보조 텍스트

- 입력창 아래 `caption`, `text-muted`.
- 오류 시 같은 위치에서 `danger` 로 변경.

### 9.3 폼 간격

- 필드 간 세로 간격: `form-gap`.
- 같은 row 에 두 필드를 둘 때는 `card-gap` 의 절반.

### 9.4 적용

- 신환 등록 폼 (차트번호 / 이름 / 생년월일 / 연락처).
- 환자 정보 수정.
- 치료사 / 의사 등록 / 수정.
- 관리자 설정.

## 10. AI 도우미 카드 (예약 / 휴무 공통 패턴)

### 10.1 기본 상태

- 카드 헤더 좌측: "AI 예약 도우미" / "AI 휴무 도우미" (`card-title`).
- 카드 본문: 입력창 + `[AI 확인]` 버튼 (가로 정렬, 카드 패딩 안에서).
- 하단: 예시 문구 (`caption`, `text-muted`).
- 결과 영역은 펼쳐지기 전까지 표시되지 않음.

### 10.2 펼친 상태

- 입력창 영역과 결과 영역 사이 분리 보더 1px `border` (선택).
- 결과 영역 상단 우측에 `[접기]` `[초기화]`.
- 결과 영역 안의 환자 후보 / 치료항목 후보 / 신환 등록 / 최종 승인 카드는 **개별 sub-card** 로 표시 (카드 안의 카드).
- sub-card 그림자는 더 약하게 (혹은 보더만으로 구분).

### 10.3 접힌 상태

- 입력창은 유지.
- 진행 중 상태 badge 표시 (예: `승인 대기` / `환자 선택 필요`).
- 우측에 `[펼치기]`.

### 10.4 전이 (트랜지션)

- 펼침 / 접힘은 짧은 height transition (~150ms) — 과한 애니메이션 금지.
- 카드 자체 위치 이동 / 큰 fade 효과 금지 (실무용 안정성).

---

## 11. 환자 후보 / 치료항목 후보 / 신환 등록 sub-card

### 11.1 환자 후보 카드

- 헤더: "후보 N"
- 본문: 차트번호 / 이름 / 생년월일 / 연락처 (라벨 + 값, 그리드 정렬).
- 우측 하단: `[이 환자 선택]` (Primary).
- 선택된 후보는 보더 `primary`, 배경 `primary-soft`.

### 11.2 치료항목 후보 카드

- 항목별 1줄: `원문 → 매칭 결과 [상태 badge]`
- 매칭 미정 항목 우측에 `[선택]` 버튼 (드롭다운 또는 별도 모달).
- 선택 전엔 [예약 등록] 비활성.

### 11.3 신환 등록 sub-card

- 폼: 차트번호 / 이름 / 생년월일 / 연락처.
- 중복 검사 결과는 폼 아래 메시지로 (§ 8).
- 승인 카드는 별도 sub-card 로 한번 더 노출 (이중 확인).

### 11.4 최종 예약 / 휴무 후보 카드

- 환자 정보 (예약) 또는 치료사 정보 (휴무) 블록.
- 예약 정보 / 휴무 정보 블록.
- 검증 결과 블록 (체크 아이콘 + 항목).
- 안내 문구 + `[취소]` `[예약 등록]` / `[휴무 등록]`.
- 승인 전: "예약 후보" / "휴무 후보" 표시. "완료" 단정 표현 금지.

---

## 12. 캘린더 (예약관리 / 휴무일관리 공통)

### 12.1 컨테이너

- 흰색 카드 안.
- 카드 헤더에 월 / 주 토글, 이전 / 다음 이동 버튼.

### 12.2 셀

- 셀 보더 얇고 연하게.
- 오늘 날짜는 `primary-soft` 배경 / 라벨 굵기.
- 휴무 / 반차 표시는 셀 안에 작은 badge.
- 치료사별 색상 도트는 작게 / 정렬 통일.
- 예약 표시는 시간 + 환자명 + 치료항목 약어 (가독성 우선).

### 12.3 충돌 / 휴무 시각화

- 휴무 시간대는 셀 배경에 사선 패턴 또는 옅은 회색 채움.
- 사용자가 즉시 식별 가능해야 함.

## 13. 관리자 / AI 로그 화면 스타일

### 13.1 관리자 설정 카드

- 한 카드 = 한 설정 그룹.
- 헤더에 그룹명 (`card-title`).
- 위험 작업 (백업 삭제, AI 로그 삭제 등): 본문 우측 액션 버튼 `Danger`, 본문에 경고 메시지 (§ 8).

### 13.2 AI 로그 표

- 시간 / 사용자 / 원본 명령 / 상태 / 실행 결과.
- 상태는 badge.
- 행 클릭 → 상세 카드 / 모달 펼침.

### 13.3 AI 로그 상세

- 원본 명령 / AI 추출 결과 / DB 검증 / 환자 후보·선택 / 치료항목 후보·선택 / 승인 / 실행 / 오류.
- JSON 원문은 접기 / 펼치기 (`[원문 보기]` 토글).
- 일반 직원이 보기 복잡하지 않게 관리자용으로 정돈.

## 14. 접근성 / 가독성

- 텍스트 명도 대비는 본문 4.5:1 이상 권장.
- 클릭 영역은 최소 32~40px 확보.
- 색만으로 의미를 전달하지 않음 (badge 텍스트도 함께).
- 키보드 포커스는 `focus-ring` 으로 명확히 표시.
- 화면 확대(브라우저 줌) 125% 까지 깨지지 않게.
