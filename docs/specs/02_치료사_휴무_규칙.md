# 02. 치료사 휴무 규칙

본 문서는 치료사 휴무 등록과, 휴무가 있는 치료사에게 들어가는 예약을 어떻게 차단하는지 정의한다.
관련 모델: [app/models/models.py](../../app/models/models.py) `EmployeeLeave`.
관련 라우터: [app/routers/api.py](../../app/routers/api.py) `GET /api/employee-leaves`, `POST /api/employee-leaves`, `POST /api/employee-leaves/bulk-set`.
관련 테스트: [tests/test_therapist_leave.py](../../tests/test_therapist_leave.py).

---

## 1. `leave_type` 값 표준

`EmployeeLeave.leave_type` 컬럼은 다음 세 가지 값만 허용한다.

| 값 | 의미 | 차단 시간대 |
|------|------|-------------|
| `"full"` | 종일 휴무 | 해당 날짜 전체 (00:00 ~ 23:59) |
| `"morning"` | 오전반차 | `start_at.time() < 12:00` |
| `"afternoon"` | 오후반차 | `start_at.time() >= 12:00` |

기준 시각은 **12:00 정각**. `start_at` 의 시각 부분(분/초 무시)을 기준으로 비교.

> 현재 모델 ([app/models/models.py:47](../../app/models/models.py:47)) 은 `default="full"` 로 코드에 `"full"` 만 명시되어 있다.
> `"morning"` / `"afternoon"` 은 본 spec 으로 표준 정의하며, 시드 데이터와 테스트에서 사용한다.

---

## 2. 예약 차단 규칙

치료사가 특정 날짜에 휴무가 있으면, 해당 치료사로 그 날짜에 예약을 새로 생성하거나 수정해 옮기는 동작은 차단되어야 한다.

| 휴무 유형 | 예약 시각 | 결과 |
|------|------|------|
| `full` (종일) | 그 날 어떤 시간이든 | ❌ 차단 |
| `morning` (오전반차) | `< 12:00` | ❌ 차단 |
| `morning` (오전반차) | `>= 12:00` | ✅ 허용 |
| `afternoon` (오후반차) | `>= 12:00` | ❌ 차단 |
| `afternoon` (오후반차) | `< 12:00` | ✅ 허용 |

차단 대상은 **`therapist_id` 로 지정된 치료사**. `therapist_id == None` 인 미배정 예약은 휴무 차단 대상이 아니다 (배정 시점에 다시 검증되어야 함).

---

## 3. 백엔드 강제 검증

휴무 규칙은 **프론트 표시뿐 아니라 백엔드 예약 생성 API 에서도 강제되어야 한다.**

휴무가 등록된 치료사에게 프론트를 우회한 수동 API 요청 (예: `curl -X POST /api/appointments`) 으로 예약을 넣으려 해도 차단되어야 한다.

> ⚠️ 현재 백엔드 구현 상태:
> - ❌ `create_appointment` 에서 `EmployeeLeave` 테이블 조회 없음 — **미구현**
> - 테스트 ([tests/test_therapist_leave.py](../../tests/test_therapist_leave.py)) 의 차단 검증은 `@pytest.mark.xfail` 로 표시
>
> 향후 백엔드에 차단 로직이 추가될 때:
> 1. `app/routers/api.py` 의 `create_appointment` 와 `update_appointment` 에 휴무 검사 헬퍼 추가
> 2. 헬퍼는 `(therapist_id, start_at.date())` 로 `EmployeeLeave` 조회
> 3. 조회된 leave 의 `leave_type` 에 따라 위 표대로 분기
> 4. 차단 시 `HTTPException(400, "해당 날짜에 ... 인 치료사입니다.")` 한국어 메시지

---

## 4. 휴무 등록 API 흐름

| 엔드포인트 | 메서드 | 역할 |
|------|------|------|
| `/api/employee-leaves` | GET | 기간별 휴무 목록 |
| `/api/employee-leaves` | POST | 단건 등록 |
| `/api/employee-leaves/{lid}` | DELETE | 단건 삭제 |
| `/api/employee-leaves/bulk-set` | POST | 일괄 설정 |

휴무 등록 자체는 자유롭게 가능 (이미 예약이 있어도 등록은 허용 — 운영자가 보고 후속 조치). 차단은 **예약 생성/수정 시점**에서만 발생.

[tests/harness/seed_data.py](../../tests/harness/seed_data.py) 가 테스트 DB 에 멱등으로 다음 휴무를 시드한다.

|--------|------|
| 김테스트치료사 | `full` (종일) |

휴무 날짜는 테스트 시작 시점의 다음 영업일 또는 고정 미래 날짜 (구현에서 결정).
