# 05. 야간당직 시간 규칙

본 문서는 **야간당직**(`duty_type='night'`)의 퇴근시각 기록과 초과근무(야간당직) 시간 집계 방식을 정의한다.
관련 모델: [app/models/models.py](../../app/models/models.py) `EmployeeDuty.end_time`.
관련 라우터: [app/routers/api.py](../../app/routers/api.py) `GET/POST /api/employee-duties`, `.../bulk-set`, `.../bulk-add`, `_duty_overtime_minutes`.
관련 설정: `config.duty_baseline_end_time` (관리자 → 운영 시간 → 야간당직 기준).
관련 테스트: [tests/test_employee_duties.py](../../tests/test_employee_duties.py).

---

## 1. 기준 퇴근시간 (baseline)

야간당직 시간은 **기준 퇴근시간 이후 실제 퇴근한 만큼**만 집계한다.

- 기준값 = `config.duty_baseline_end_time` (기본 `"18:30"`). 운영 종료 `close_time` 과 **별개**로 관리한다.
- 관리자 설정에서 `HH:MM` 으로 변경. 빈값 저장 시 기본 `18:30` 복원, 형식 오류는 `400`.
- 기준값은 정책(설정)이므로 나중에 바꾸면 과거 당직의 집계도 **새 기준으로 재계산**된다.

---

## 2. `end_time` (실제 퇴근시각)

`EmployeeDuty.end_time` = `"HH:MM"` 문자열, nullable.

- **야간당직 전용.** 아침당직(`morning`)은 미사용(NULL) — 시간 개념 없음.
- 직원별·날짜별로 개별 기록 (같은 밤 여러 명이면 각자 다른 퇴근시각 가능).
- 빈값/NULL 이면 그 당직의 야간시간은 0.
- 입력 형식은 `HH:MM` 또는 빈값만 허용, 그 외는 백엔드에서 `400` (프론트 우회 요청도 차단).

---

## 3. 초과분 계산 `_duty_overtime_minutes(end_time, baseline)`

반환값은 **분(minutes)**. `end` = end_time 의 분환산, `base` = baseline 의 분환산.

| 조건 | 결과(분) | 의미 |
|------|----------|------|
| `end_time` 빈값/형식오류 | `0` | 미입력/아침당직 |
| `end >= base` | `end - base` | 같은 저녁 초과근무 |
| `end < base` 이고 `end < 12:00` | `end - base + 1440` | 자정 넘겨 새벽 퇴근 |
| `end < base` 이고 `end >= 12:00` | `0` | 기준보다 이른 오후 = 오입력 방지 |

예 (기준 18:30):
- `21:00` → 150분 (2시간 30분)
- `01:00` → 390분 (6시간 30분, 자정 넘김)
- `18:30` → 0분
- `17:00` → 0분 (기준 이전)

---

## 4. 집계 표시

집계 계산은 **백엔드**가 담당한다 — `GET /api/employee-duties` 응답의 각 행에 서버가 계산한
`overtime_minutes` 를 포함한다. 프론트(`renderDutyMonthlySummary`)는 직원별로 **합산·표시만** 한다.

- 캘린더에 보이는 달·현재 하위탭 유형 기준으로 직원별 `overtime_minutes` 합계를 낸다.
- 기존 "당직 횟수(N회)" 요약 칩에 시간을 **함께** 표기: `이름 · N회 · N시간 N분`.
- 야간당직 탭에서 시간>0 일 때만 시간 부분을 표시. 아침당직 탭은 "N회"만.

---

## 5. 아침당직 (morning) 불변

아침당직에는 퇴근시간·시간집계를 적용하지 않는다. `end_time` 미입력, `overtime_minutes=0`,
요약 칩은 "N회"만. 본 기능은 야간당직에 한정된다.
