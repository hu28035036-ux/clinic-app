# 03. 완료 카운트 / 통계 규칙

본 문서는 예약이 완료(승인) 처리되었을 때 환자별·항목별 카운트가 어떻게 누적되는지, 그리고 통계 API 가 어떤 기준으로 집계하는지 정의한다.
관련 모델: [app/models/models.py](../../app/models/models.py) `Appointment`, `Treatment`, `PatientTreatmentCount`, `ManualCount`.
관련 라우터: [app/routers/api.py](../../app/routers/api.py) `POST /api/appointments/{aid}/approve`, `GET /api/stats/*`.
관련 테스트: [tests/test_stats_counts.py](../../tests/test_stats_counts.py).

---

## 1. "완료" 의 정의

`Appointment.status` 의 가능한 값:
- `"reserved"` — 예약됨 (기본값)
- `"approved"` — **완료 (= 원무과 승인)**
- `"canceled"` — 취소

완료 통계는 `status == "approved"` 만 집계한다. `"reserved"` 는 예약 통계에만, `"canceled"` 는 어떤 통계에도 포함되지 않는다.

승인 엔드포인트: `POST /api/appointments/{aid}/approve` ([app/routers/api.py:1770](../../app/routers/api.py:1770) 부근).
승인 시 `_bump_patient_count` 를 호출해 `PatientTreatmentCount.done_count` 를 항목별로 +1.

---

## 2. 항목별 완료 카운트 — 1건 = 1카운트

### 핵심 규칙

**도수치료 항목 1건 완료 = 해당 코드의 완료 카운트 +1.**

| 완료된 예약의 `treatment_codes` | 카운트 증가 |
|------|------|
| `["manual30"]` | manual30 +1 |
| `["manual60"]` | manual60 +1 |
| `["manual30", "manual60"]` | manual30 +1, manual60 +1 |
| `["manual30", "eswt"]` | manual30 +1, eswt +1 |

### 절대 금지

**`manual60` 을 2 카운트로 계산하지 않는다.**

- 과거에는 `Treatment.count_increment=2` 로 manual60 을 두 카운트로 누적한 적이 있었으나, 현 정책은 **모든 항목 1건 = 1카운트**.
- 시드 (`app/models/constants.py:14-21`) 에서 manual30, manual60 모두 `count_increment=1`.
- 이 정책을 되돌리는 변경은 [CLAUDE.md](../../CLAUDE.md) 의 "절대 금지" 항목.

### 향후 항목 추가 (예: `manual90`)

새 도수치료 항목이 추가되어도 동일하게 **1건 = 1카운트** 로 별도 항목으로 집계한다.
- `manual90` 추가 시 `Treatment` 행을 생성하고 `count_increment=1` 로 둔다.
- 통계 API 에서 자동으로 별도 항목으로 노출된다 (코드별 그룹화).

---

## 3. 체외충격파(`eswt`) 별도 집계

`eswt` 는 도수치료(`manual*`) 와 **별도 집계** 한다.

- 일반 예약 흐름 (예약 → 승인) 으로 들어온 `eswt` 는 `PatientTreatmentCount` 에 누적되며 통계에서 도수치료와 같은 방식으로 집계.
- 단, "당일 내방 환자 수동 카운트" 는 `ManualCount` 테이블에서 별도 합산 ([app/migrations/m006_manual_counts.py](../../app/migrations/m006_manual_counts.py) 참조).
- 통계 API `/api/stats/aggregate` 등은 ManualCount 와 예약 기반 카운트를 합산해서 보여줌 ([app/routers/api.py:3367 부근](../../app/routers/api.py:3367)).

도수치료 통계와 체외충격파 통계가 한 셀에 섞여 더해지지 않도록 코드(treatment_code) 단위로 정확히 분리되어야 한다.

---

## 4. 취소된 예약은 통계에서 제외

`status == "canceled"` 인 예약은:
- 완료 카운트에 포함되지 않는다 (애초에 승인 안 됐으므로).
- 예약 기준 통계에서도 제외하는 것이 일반적이지만, 일부 통계는 "예약 시도 횟수" 를 보여줄 수 있다 — 그 경우 의미가 명확히 표시되어야 한다.
- 완료 기준 통계와 예약 기준 통계가 한 표에 섞여 보이지 않도록 기준 라벨을 붙인다.

---

## 5. 완료 기준 ↔ 예약 기준 통계 분리

통계 API 에는 두 가지 기준이 존재할 수 있다.

| 기준 | 의미 | 사용처 |
|------|------|--------|
| 완료 기준 | `status == "approved"` 만 집계 | 매출·인센티브 계산, 카운트 집계 |
| 예약 기준 | `status in ("reserved", "approved")` | 예상 매출, 일정표 표시 |

이 두 기준이 한 통계 표에 섞이지 않도록 코드 레벨에서 명확히 분리되어야 한다.
관련 엔드포인트:
- `GET /api/stats/by-therapist` — 예약/완료 기준 모두 노출
- `GET /api/stats/manual-by-therapist` — 도수치료 항목별
- `GET /api/stats/aggregate` — 완료 기반 + ManualCount 합산
- `GET /api/stats/daily-by-therapist` — 일별×치료사별
- `GET /api/stats/summary` — 상단 카드 요약

## 6. 테스트 커버리지

[tests/test_stats_counts.py](../../tests/test_stats_counts.py) 가 다음을 검증한다.

- manual30 1건 승인 → manual30 done_count 정확히 +1
- manual60 1건 승인 → manual60 done_count 정확히 +1 (NOT +2)
- manual30 + manual60 동시 승인 → 각각 +1
- manual90 (새 Treatment 추가) 승인 → manual90 별도 +1
- eswt 승인 → 도수치료 통계와 별도 집계
- 취소된 예약은 done_count 증가 없음
