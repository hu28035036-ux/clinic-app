================================================================
  도수치료예약 (병원 예약 관리) - 사용법 및 안내사항
  버전: 1.2.2 (2026-04-19)
================================================================

┌──────────────────────────────────────────────────────────────┐
│ 1. 프로그램 실행 방법                                         │
└──────────────────────────────────────────────────────────────┘

[신규 설치]
  1) 받으신 "도수치료예약.zip" 을 원하는 위치에 압축 해제
     예) C:\도수치료예약\
  2) 압축 해제한 폴더 안의 "도수치료예약.exe" 를 더블클릭
  3) 처음 실행되면 자동으로:
     - 메인 서버가 시작되고
     - 기본 브라우저가 열려 http://127.0.0.1:8000 로 접속
     - 바탕화면에 "병원 예약 관리" 바로가기 자동 생성

[다음 번 실행부터]
  - 바탕화면의 "병원 예약 관리" 바로가기 더블클릭 또는
  - "도수치료예약.exe" 를 직접 실행
  - 이미 실행 중이면 브라우저만 새로 열림

[다른 PC·모바일에서 접속]
  - 프로그램이 실행된 PC 의 IP 주소 확인
    (실행 시 명령창에 "다른 PC/폰: http://xxx.xxx.xxx.xxx:8000" 형태로 표시)
  - 같은 Wi-Fi/LAN 망의 다른 기기 브라우저에서 해당 주소로 접속


┌──────────────────────────────────────────────────────────────┐
│ 2. 최초 실행 전 확인사항                                     │
└──────────────────────────────────────────────────────────────┘

[필수 확인]
  □ Windows 10 이상 (64bit 권장)
  □ 8000 포트가 다른 프로그램에 사용 중이지 않아야 함
  □ 백신/방화벽이 실행 파일 차단하지 않는지 확인
    (차단 시 "예외 추가" 또는 "허용" 선택)

[권장 사항]
  □ 관리자 비밀번호 즉시 변경 (초기값: 5036 또는 설치 시 지정값)
    → 프로그램 실행 → ≡ 관리자 탭 → 시스템 → 관리자 비밀번호 변경
  □ 문자나라 연동 설정 입력
    → ≡ 관리자 → ✉ 문자나라
    → 아이디 / 2차 비밀번호 / 발신번호 입력
    → API URL 은 기본값 그대로 사용 (https://munjanara.co.kr/send.sys)
  □ 운영시간·슬롯 분 단위 설정
    → ≡ 관리자 → 시스템 → 운영 시간 카드


┌──────────────────────────────────────────────────────────────┐
│ 3. 데이터 저장 위치 (중요!)                                  │
└──────────────────────────────────────────────────────────────┘

  프로그램 실데이터는 아래 폴더에 저장됩니다:

  C:\Users\(사용자이름)\AppData\Roaming\도수치료예약\
    ├── clinic.db          ← 환자/예약/직원 DB (절대 삭제 금지)
    ├── config.json        ← 운영 설정 (포트/운영시간 등)
    ├── schema_version.txt ← DB 스키마 버전
    └── backups\           ← 자동 백업 보관소

  ⚠ 프로그램 폴더(도수치료예약\*)에는 데이터가 저장되지 않습니다.
    → 프로그램 폴더를 다른 곳으로 옮기거나 삭제해도 데이터는 유지됩니다.
    → 업데이트 시 프로그램 폴더만 교체하면 데이터 그대로 유지.


┌──────────────────────────────────────────────────────────────┐
│ 4. 데이터 백업 관련 주의사항                                 │
└──────────────────────────────────────────────────────────────┘

[자동 백업]
  - 프로그램이 자동으로 backups\ 폴더에 DB 사본을 보관합니다.
  - ≡ 관리자 → 시스템 → "자동 백업" 카드에서 주기/보관 개수 조정 가능.

[수동 백업 (권장)]
  - 주기적으로 아래 폴더를 USB 또는 클라우드에 복사 보관:
    C:\Users\(사용자이름)\AppData\Roaming\도수치료예약\
  - 특히 대량 데이터 import 전·업데이트 전에는 반드시 수동 백업.

[복원]
  - 프로그램 종료 → backups\ 에서 원하는 .db 파일을 clinic.db 로 덮어쓰기
  - 프로그램 재시작

[⚠ 외부 클라우드 동기화 주의]
  - clinic.db 는 개인정보(환자명/생년월일/연락처) 포함
  - Google Drive / Dropbox 등 공용 클라우드에 실데이터 저장 시
    개인정보보호법 검토 필수
  - AppData 폴더는 기본적으로 OS 백업 범위 밖 → 안전하게 유지됨


┌──────────────────────────────────────────────────────────────┐
│ 5. 종료 전 확인할 점                                         │
└──────────────────────────────────────────────────────────────┘

  □ 브라우저에서 저장 버튼을 누른 내용이 반영됐는지 확인
  □ 진행 중인 문자 발송 / 데이터 import 등 배치 작업 완료 확인
  □ 명령창(콘솔) 에서 오류 메시지 없었는지 간단히 훑기
  □ 프로그램 종료 = 콘솔 창 닫기 또는 Ctrl+C
    (브라우저 창 닫기만으로는 서버가 안 꺼짐)


┌──────────────────────────────────────────────────────────────┐
│ 6. 문제 발생 시 기본 점검                                    │
└──────────────────────────────────────────────────────────────┘

[브라우저가 안 열림 / 접속 안 됨]
  1) 명령창 확인 — "http://127.0.0.1:8000" 주소 표시됐는지
  2) 수동으로 브라우저에 127.0.0.1:8000 입력
  3) 방화벽/백신에서 exe 차단 여부 확인
  4) 8000 포트 다른 프로그램과 충돌 여부 확인 (드물지만)

[환자/예약 데이터가 안 보임]
  1) 프로그램 종료 후 재시작
  2) AppData\도수치료예약\clinic.db 파일 존재 확인
  3) 없거나 0바이트면 backups\ 에서 최신 .db 복원

[문자 발송 실패]
  1) ≡ 관리자 → 문자 로그 (또는 콘솔 로그) 확인
  2) code=2 → 아이디/2차비번 오류
     code=9 → 발신번호 미등록 또는 포인트 부족
  3) ≡ 관리자 → ✉ 문자나라 설정 재확인

[업데이트 후 정상 동작 안 함]
  1) 기존 AppData\도수치료예약\ 폴더 그대로 유지되었는지 확인
  2) 새 프로그램 폴더에 제대로 덮어씌워졌는지 확인
  3) "기존 설치 컴퓨터 업데이트 방법.txt" 참고하여 재시도

[관리자 비밀번호를 잊어버림]
  - AppData\도수치료예약\config.json 의 admin_password_hash 값 삭제 후 재시작
    → 초기 비밀번호(기본값) 로 리셋됨
