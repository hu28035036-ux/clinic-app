================================================================
  도수치료예약 - 기존 설치 PC 업데이트 가이드
  버전: 1.2.1 으로 업데이트
================================================================

이 문서는 이전 버전(예: 1.0.0)이 이미 설치된 PC 에
새 버전(1.2.1)을 적용하는 방법입니다.

※ 환자/예약 DB 는 프로그램 폴더 밖(AppData)에 저장되므로
  아래 절차만 지키면 기존 데이터는 100% 유지됩니다.


┌──────────────────────────────────────────────────────────────┐
│ [업데이트 절차 요약 · 5분 소요]                              │
└──────────────────────────────────────────────────────────────┘

  1단계. 프로그램 완전 종료
  2단계. (선택·권장) 데이터 폴더 수동 백업
  3단계. 기존 프로그램 폴더 → 다른 이름으로 변경 (롤백용)
  4단계. 새 배포본 ZIP 압축 해제
  5단계. 프로그램 실행 + 데이터 유지 확인
  6단계. 문제 없으면 구 폴더 삭제

각 단계 상세 설명 ↓


┌──────────────────────────────────────────────────────────────┐
│ 1단계. 프로그램 완전 종료                                    │
└──────────────────────────────────────────────────────────────┘

  □ 브라우저 탭 닫기 (병원 예약 관리 열린 탭 모두)
  □ 작업표시줄의 검은 명령창(또는 도수치료예약.exe 프로세스) 닫기
    - 안 보이면 Ctrl + Shift + Esc → 작업관리자 → "도수치료예약"
      프로세스 찾아서 "작업 끝내기"
  □ Google Drive 등 파일 싱크 프로그램 사용 중이면
    잠시 "일시정지" 상태로 (업데이트 중 싱크 충돌 방지)


┌──────────────────────────────────────────────────────────────┐
│ 2단계. 데이터 폴더 수동 백업 (선택 · 강력 권장)              │
└──────────────────────────────────────────────────────────────┘

  업데이트 절차상 AppData 는 건드리지 않지만,
  만약의 사고 대비해 아래 폴더를 USB 또는 다른 위치에 복사:

  [위치] 파일 탐색기 주소창에 입력:
         %APPDATA%\도수치료예약

  [권장 복사 방법]
    - 폴더 우클릭 → "복사"
    - 바탕화면 등에 "도수치료예약_백업_YYYYMMDD" 폴더로 붙여넣기


┌──────────────────────────────────────────────────────────────┐
│ 3단계. 기존 프로그램 폴더를 백업용 이름으로 변경              │
└──────────────────────────────────────────────────────────────┘

  현재 설치된 프로그램 폴더 (예: C:\도수치료예약) 를
  롤백용으로 이름만 변경:

    C:\도수치료예약        →  C:\도수치료예약_old_20260418

  ※ 이렇게 하면 새 버전 설치 후 문제 발생 시
    바로 이전 버전으로 되돌릴 수 있습니다.
  ※ 이름 변경만 하세요. 삭제하지 마세요.


┌──────────────────────────────────────────────────────────────┐
│ 4단계. 새 배포본 압축 해제                                   │
└──────────────────────────────────────────────────────────────┘

  □ 새 배포본 ZIP 파일 ("도수치료예약_v1.2.1.zip" 등) 을
    구 폴더와 같은 위치에 압축 해제
  □ 해제된 폴더명이 기존과 같은 "도수치료예약" 으로
    되도록 (압축 프로그램 기본 설정이면 자동)

    결과 예시:
      C:\도수치료예약_old_20260418    ← 이전 버전 (롤백용 보관)
      C:\도수치료예약                 ← 새 버전 1.2.1

  ※ 두 폴더가 같은 부모 폴더에 나란히 있어야 합니다.


┌──────────────────────────────────────────────────────────────┐
│ 5단계. 프로그램 실행 + 데이터 유지 확인                      │
└──────────────────────────────────────────────────────────────┘

  □ 새 폴더 안의 "도수치료예약.exe" 실행
  □ 브라우저 자동 실행 확인
  □ 아래 3가지 데이터가 그대로 보이는지 확인:

    [확인 1] 환자 탭
       - 기존 등록된 환자들이 그대로 보임
    [확인 2] 예약 현황 탭
       - 이전 예약들이 그대로 보임
    [확인 3] 관리자 → ≡ 시스템 → "• 버전 / 업데이트" 카드
       - 현재 버전: v1.2.1 표시됨
       - 데이터 폴더 경로:
         C:\Users\(사용자)\AppData\Roaming\도수치료예약

  ✅ 3가지 모두 OK → 업데이트 성공


┌──────────────────────────────────────────────────────────────┐
│ 6단계. 구 폴더 정리                                          │
└──────────────────────────────────────────────────────────────┘

  업데이트 후 며칠 정상 운영했다면
  "도수치료예약_old_YYYYMMDD" 폴더는 삭제해도 됩니다.

  (AppData 의 데이터는 영향 없음)


┌──────────────────────────────────────────────────────────────┐
│ [중요] 절대 덮어쓰면 안 되는 파일                            │
└──────────────────────────────────────────────────────────────┘

  아래 폴더는 업데이트 중 절대 삭제/이동/덮어쓰기 금지:

  %APPDATA%\도수치료예약\
    ├── clinic.db          ← 환자/예약 DB (실운영 데이터)
    ├── config.json        ← 설정 · 관리자 비밀번호 해시
    ├── schema_version.txt
    └── backups\           ← 자동 백업들

  ⚠ 이 폴더는 프로그램 폴더와 완전히 분리되어 있어서
    프로그램 폴더를 통째로 교체해도 이 폴더는 건드려지지 않습니다.
    (업데이트 시 별도 주의 필요 없음 — 그냥 두기만 하면 됨)


┌──────────────────────────────────────────────────────────────┐
│ [유지/교체 정리]                                             │
└──────────────────────────────────────────────────────────────┘

  ■ 교체 (새 버전으로 덮어쓰기)
      C:\도수치료예약\*             ← 프로그램 폴더 전체
      (exe / _internal / 라이브러리 등)

  ■ 유지 (그대로 두기, 건드리지 말 것)
      %APPDATA%\도수치료예약\*       ← 데이터 폴더 전체


┌──────────────────────────────────────────────────────────────┐
│ 업데이트 후 처음 확인할 항목                                 │
└──────────────────────────────────────────────────────────────┘

  □ 버전 표시: ≡ 관리자 → 시스템 → 버전 카드 = v1.2.1
  □ 기존 환자/예약 데이터 정상 표시
  □ 문자 발송 테스트 (본인 번호로 1건)
  □ 새 예약 등록·수정 정상 동작
  □ 예약현황표 드래그 이동 정상 동작
  □ AI 데이터변환 탭 진입 가능 (관리자 인증 필요)


┌──────────────────────────────────────────────────────────────┐
│ 문제 발생 시 롤백 방법                                       │
└──────────────────────────────────────────────────────────────┘

  1) 새 프로그램 종료
  2) 새 "도수치료예약" 폴더를 "도수치료예약_failed_YYYYMMDD" 로 이름 변경
  3) "도수치료예약_old_YYYYMMDD" (이전 버전) 을
     "도수치료예약" 으로 이름 복원
  4) 구 버전 exe 실행 → 정상 동작 확인

  ※ AppData 의 DB는 버전과 상관없이 동일하게 읽힙니다.
    단, 새 버전에서 DB 스키마를 변경했다면 구 버전에서 경고 가능
    (v1.0.0 → v1.2.1 업데이트는 스키마 하위 호환이라 OK)


┌──────────────────────────────────────────────────────────────┐
│ 여러 PC 동시 업데이트 시                                     │
└──────────────────────────────────────────────────────────────┘

  - 원무과 PC, 치료실 PC 등 여러 대 사용 중이면
    한 PC 씩 순차 업데이트 권장
  - 메인 서버 PC 를 먼저 업데이트하고
    서브 PC 들이 정상 접속되는지 확인 후 다음 PC 진행
  - 같은 네트워크 안에서 버전 혼용은 최대 몇 시간 내로


================================================================
  문의 / 오류 제보 시 준비할 정보
================================================================

  1) 현재 버전: ≡ 관리자 → 시스템 → 버전 카드 화면 캡처
  2) 증상 설명: 어떤 작업 후 무엇이 안 되는지
  3) 브라우저 개발자 도구(F12) 콘솔 오류 메시지 (있으면)
  4) 명령창 오류 메시지 (있으면)
  5) AppData\도수치료예약\backups\ 최신 백업 파일 확보 여부
