# 데이터 백업 (backup)

## 적용 상황
- 평상시: 자동 백업이 동작하지만, 중요한 변경 직전에 수동 백업을 추가로 받고 싶을 때
- 업데이트 직전: 시스템이 자동으로 백업 스냅샷을 만들지만 한 번 더 받고 싶을 때
- 사고 복구: 잘못 삭제한 데이터를 복구하기 위해 백업 파일을 확인할 때

## 위치
- 백업 폴더 경로는 메인 화면 → 관리자 탭 → "버전 / 업데이트" 박스의 "백업 폴더" 항목에서 확인할 수 있다.
- 일반적으로 사용자 데이터 폴더(%APPDATA%\도수치료예약\backups\) 아래에 저장된다.

## 자동 백업
- 프로그램 시작 시 그날 백업이 없으면 1회 자동 백업이 만들어진다.
- 설정된 주기마다 자동 백업이 추가된다.
- 보관 개수를 초과하면 가장 오래된 파일이 자동으로 삭제된다.
- 업데이트 설치 직전에는 별도로 자동 백업 스냅샷이 생성된다 — 업데이트가 실패해도 이전 상태로 복구 가능.

## 수동 백업
1. 관리자 탭의 백업 관련 영역에서 "지금 백업" 버튼을 누른다.
2. 잠시 후 새 백업 파일이 백업 폴더에 추가된다.
3. 파일명은 clinic_YYYYMMDD_HHMMSS.db 형식이다.

## 복원
- 사고로 데이터가 손상됐을 때, 관리자 화면의 "최근 백업으로 복원" 기능으로 가장 최근 백업 파일을 현재 DB에 덮어쓸 수 있다.
- 복원 시 현재 DB 의 변경분은 사라진다 — 복원 직전 한 번 더 수동 백업을 만들어 두는 것을 권장한다.

## 주의
- 백업 폴더 위치는 절대 변경하지 않는다. 시스템이 정해진 경로를 사용한다.
- 백업 파일을 외부로 옮길 때는 환자 개인정보가 포함되어 있음을 인지하고 안전하게 보관한다.
- 업데이트는 프로그램 폴더(exe + _internal)만 교체하므로 데이터 폴더의 DB 와 백업은 그대로 유지된다.
