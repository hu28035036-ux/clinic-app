# 예약문자 작성 (sms_compose)

## 적용 상황
- 환자에게 예약 안내, 변경 안내, 리마인더 등 개별 문자를 보낼 때
- 내일 예약 환자에게 일괄 안내 문자를 보낼 때

## 위치
- 메인 화면 상단 탭에서 "예약 문자" 탭 선택

## 절차 — 개별 문자 발송
1. "예약 문자" 탭 → "개별 문자 발송" 섹션을 사용한다.
2. 환자 검색창에서 환자명 또는 차트번호로 검색해서 대상 환자를 선택한다.
3. 연락처가 자동 입력되는지 확인한다. 비어 있으면 직접 입력한다.
4. 템플릿을 선택하면 본문이 자동 채워진다. 변수(예: {환자명}, {예약일시})는 자동 치환된다.
5. 본문을 검토·수정한 뒤 "발송" 버튼을 누른다.
6. 발송 결과(성공/실패)를 확인한다.

## 절차 — 내일 예약 일괄 안내
1. "예약 문자" 탭 → "내일 예약 일괄 안내" 섹션을 연다.
2. "대상 불러오기"를 눌러 내일 예약 환자 목록을 받는다.
3. 일괄 적용할 템플릿을 선택한다. 환자별 변수는 자동 치환된다.
4. 보낼 환자만 체크박스로 선택한다.
5. "선택한 환자에게 발송"을 누르면 일괄 발송된다.

## AI 초안 사용
- 본문 작성이 막힐 때 "AI 초안 받기" 버튼을 누르면 톤(친근/정중) 별로 초안을 받을 수 있다.
- AI 초안은 발송 직전 사람이 반드시 검토·수정해야 한다. AI 가 직접 발송하지 않는다.
- AI 기능이 꺼져 있거나 API Key 가 없으면 초안 버튼이 안내 문구를 표시한다 — 관리자 → AI 설정에서 활성화한다.

## 발송 전 자동 점검
- 발송 직전 시스템이 누락 정보(전화번호, 본문 비어 있음, 미치환 변수, CP949 깨짐 문자 등)를 자동 점검한다.
- 차단 항목(blocker)이 있으면 발송이 막힌다. 안내 메시지를 보고 본문/연락처를 수정한다.
- 경고(warning)는 발송 가능하지만 사용자 확인이 필요하다 — 예: 치료사 미배정, 동일 본문 최근 발송 이력.

## 금지·주의
- 본문에 환자 외 다른 환자의 이름·연락처를 섞어 쓰지 않는다.
- 본문에서 변수가 치환되지 않은 채(예: "{환자명}") 발송되지 않도록 사전 점검 메시지를 무시하지 않는다.
- 발송 실패 메시지가 나오면 "문자나라 오류 대응" 매뉴얼을 참고한다.
