# 문자나라 오류 대응 (munjanara_error)

## 적용 상황
- 예약 문자 발송 시 "발송 실패" 또는 "문자나라 연동 주소 오류" 같은 메시지가 나올 때
- HTTP 404, 401, 타임아웃 등 외부 발송 게이트웨이 응답이 비정상일 때

## 일반 절차
1. 한 건 다시 발송해 본다. 일시적 네트워크 문제는 재시도로 해결되는 경우가 많다.
2. 같은 오류가 반복되면 즉시 발송을 멈추고 아래 점검표를 확인한다.

## 점검표 — 관리자 → 문자나라 연동 설정
1. 메인 화면 → 관리자 탭 → "문자나라 연동 설정"을 연다.
2. 다음 항목을 확인한다:
   - API URL: 문자나라 고객센터에서 안내받은 정확한 발송 URL 인지 확인. URL 오타·접속 경로 누락이 가장 흔한 원인이다.
   - API Key / 발신자 정보: 비어 있거나 만료되지 않았는지 확인.
   - 발신 번호 등록 여부: 문자나라 측에 등록된 발신 번호인지 확인.
3. 변경 후 "저장"을 누르고, 한 건 테스트 발송을 다시 시도한다.

## 오류별 대응
- HTTP 404: API URL 오타 가능성이 가장 높다. 문자나라 측에서 안내받은 정확한 발송 URL 로 다시 입력한다.
- HTTP 401 / 인증 오류: API Key 만료 또는 잘못된 키. 문자나라 고객센터에서 키를 재발급 받아 입력한다.
- 타임아웃 / 네트워크 오류: 인터넷 연결 상태 확인 후 재시도. 지속되면 문자나라 측 장애 가능성.
- 본문 인코딩 오류: 본문에 이모지, 한자, 특수문자가 섞여 있는지 확인. CP949 인코딩 안 되는 문자는 시스템이 사전 점검에서 차단하지만, 통과한 후에도 게이트웨이가 거절할 수 있다.

## 금지·주의
- 같은 오류가 반복되는데 무시하고 일괄 발송을 강행하지 않는다 — 문자 비용만 소모되고 발송이 안 된다.
- API Key 같은 민감 정보를 다른 사람에게 평문으로 전달하지 않는다. 시스템 화면에서는 키가 마스킹되어 표시된다.
- 환자 측 문제(차단 번호, 결번 등)는 문자나라 발송 게이트웨이 오류와 다르다. 한 명만 실패하면 환자 연락처를 먼저 확인한다.
