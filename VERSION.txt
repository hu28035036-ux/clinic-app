  도수치료예약 (병원 예약 관리 프로그램)

[주요 변경 (v1.3.56) — 기록 탭 메모 + 야간당직 퇴근시간·시간 집계]
  - [신규] 기록 탭에 "메모" 칸 추가 (차트번호·성함·직원 옆, 선택 입력, 최대 200자).
    · 추가 폼·목록 표·수정 창에서 입력/확인. 차트번호 또는 성함은 여전히 필수.
  - [신규] 야간당직에 직원별 "퇴근시각"을 날짜마다 입력 (같은 날 여러 명 각자 다르게).
    · 날짜 클릭 설정 창: 당직 직원 체크 옆 퇴근시각 칸.
    · "야간당직 추가"(직원 1명·여러 날짜) 창: 선택한 모든 날짜 공통 퇴근시각 일괄 입력.
  - [신규] 기준 퇴근시간 이후 근무분을 "야간당직 시간"으로 집계 →
    "과별 당직 횟수" 요약에 직원별 시간 병기 (예: 홍길동 5회 · 12시간 30분).
    · 기준 퇴근시간 = 관리자 → 시스템 → 운영 시간의 "야간당직 기준"(기본 18:30,
      운영 종료와 별개). 자정 넘김 퇴근도 정확 계산. 아침당직은 미적용.
  - 검증: 기록 메모 왕복 + 당직 end_time 왕복·초과분 계산 테스트 + ruff.
    DB 마이그레이션 m044(record_entries.memo) + m045(employee_duties.end_time)
    — 컬럼 추가만·보존·멱등.

[이전 변경 (v1.3.55) — 휴무 카운트·과별 보기 + 집계→정산 자동 반영 + 집계 칸 축소]
  - [추가] 휴무일 관리에 "과별 휴무 일수" 요약 (당직 요약과 같은 방식).
    · 달력에 보이는 달 기준으로 직원별 휴무 일수를 과별로 묶어 표시.
      휴무가 있는 직원만 많은 순 정렬, 달을 넘기면 그 달 기준으로 갱신.
  - [추가] 휴무일 관리에 "과" 선택 필터 (당직과 동일한 체크박스 다중 선택).
    · 선택한 과의 직원 휴무만 달력·요약에 표시. 최소 1개 과는 항상 유지.
    · 처음 열면 전체 과가 선택된 상태 — 기존처럼 전 직원 휴무가 다 보임.
  - [변경] 집계 → 정산 반영이 자동으로 바뀜 ("정산 반영" 버튼 제거).
    · 정산 탭에서 기간을 조회하면 그 기간 집계가 자동 반영되어 나옴.
      집계 칸(walk-in 포함) 수정분도 정산 조회 시 바로 반영.
    · ⚠ 정산 금액은 조회하는 시점의 수가·인센티브로 계산됨 — 수가를 바꾸면
      과거 기간 정산 금액도 새 수가로 다시 계산됨 (항상 최신 기준).
  - [개선] 집계 표의 카운팅 칸을 가로·세로로 축소해 한 화면에 더 많이 표시.
    · 칸 60×34 → 46×26, 입력칸 52×36 → 40×22 (칸 여백도 축소).
  - 검증: 전체 회귀 2,318건 + 격리 환경 브라우저 점검 + ruff + node.
    DB·마이그레이션 변동 없음 (정산 반영 API 에 로그 생략 옵션만 추가).

[이전 변경 (v1.3.53) — 점심당직 + 당직 캘린더 과 다중 선택]
  - [신규] 직원 탭 당직 메뉴에 "점심당직" 추가 (아침당직·야간당직 사이).
    · 아침/야간과 완전 독립 — 같은 직원이 같은 날 세 유형 모두 가능,
      한 유형 수정이 다른 유형에 영향 없음. DB·마이그레이션 변동 없음.
  - [개선] 당직 캘린더 과 선택 → 체크박스 드롭다운으로 여러 과 동시 선택
    (2개, 3개, 4개 등 — '전체' 없음, 최소 1개 과 항상 유지).
    · 선택 과들의 당직이 캘린더에 함께 표시, "과별 당직 횟수"도 과별 그룹 표시.
    · 선택칸: 2개까지 이름 나열, 3개+ 는 "○○ 외 N개".
  - 검증: 전체 회귀 2,318건(점심당직 테스트 신규) + 격리 브라우저 점검
    + ruff + node. 백엔드는 허용 유형 추가만, DB 무변경.

  - 검증: 전체 회귀 + 격리 브라우저 점검(열림/전환/제목 동기/바깥클릭/배지)
