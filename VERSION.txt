[주요 변경 (v1.3.3) — AI/RAG v1 후속 보강]

  🛡️ 직원 휴무 중복 차단 (m011)
     · employee_leaves(employee_id, leave_date) UNIQUE 제약 추가
     · 다중 워커 race 시 동일 직원·동일 날짜 휴무 row 중복 차단
     · 마이그레이션이 기존 중복을 (created_at DESC, id DESC) 최신
       1건만 보존하고 나머지 자동 정리 (정상 환경엔 중복 없음)
     · 사후 추적: 삭제 대상 row 정보가 stderr 로 출력
     · ⚠ 업데이트 적용 전 clinic.db 수동 백업 권장

  🛡️ AI 사용 로그 outcome 길이 확장
     · AiUsageLog.outcome String(20) → String(50)
     · "overwrite_not_acknowledged" 등 긴 코드도 truncate 없이 저장
     · SQLite VARCHAR 길이 미강제 — m008 마이그레이션 수정 없음

  🛡️ AI 상태 점검 엔드포인트 권한 분리
     · GET /api/ai/health/public 신설 (인증 불필요)
     · 일반 사용자도 SMS 탭에서 AI 상태 자체 확인 가능
     · 기존 admin 전용 /api/ai/health 그대로 유지

  🛡️ AI SDK import 진단 메시지 노출 (admin)
     · _check_sdk() 가 import 실패 사유를 짧게 보존 → admin /health 응답에
       sdk_errors: {provider: msg} 필드 추가 (public 엔드포인트엔 미노출)
     · AI 설정 박스 + 연결 테스트 alert 에 사유 inline 표시
       ("⛔ openai SDK 미설치 (ModuleNotFoundError: ...)")
     · 폴링 비용 절감 위해 module-level 캐시 (서버 재시작 시 초기화)

  🛡️ PyInstaller 번들 누락 방지 (spec)
     · 이전 v1.3.2 빌드에서 _internal/ 의 openai/anthropic 누락 사고 발생
     · 원인: spec 의 try/except 가 collect_submodules 실패를 조용히 삼킴
     · 이제 collect_submodules 가 빈 리스트면 RuntimeError 로 빌드 중단
     · collect_data_files('openai'/'anthropic') 도 datas 에 명시 추가

  ✅ 회귀 방지 테스트 +8건 (175 → 183 passed)
     · test_employee_leave_unique — 인덱스/IntegrityError/m011 단위
     · test_ai_action_leave T22b — outcome full-length 저장
     · test_ai_health_public — public/admin 권한 분리 + sdk_errors 회귀

  ⚙️ 인프라

     · spec hiddenimports 에 action_leave / date_resolver 명시
