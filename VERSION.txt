[주요 변경 (v1.3.57) — 기록 탭 메모 + 야간당직 퇴근시간·시간 집계]
  - [신규] 기록 탭에 "메모" 칸 추가 (차트번호·성함·직원 옆, 선택 입력, 최대 200자).
    · 추가 폼·목록 표·수정 창에서 입력/확인. 차트번호 또는 성함은 여전히 필수.
  - [신규] 야간당직에 직원별 "퇴근시각"을 날짜마다 입력 (같은 날 여러 명 각자 다르게).
    · 날짜 클릭 설정 창: 당직 직원 체크 옆 퇴근시각 칸.
    · "야간당직 추가"(직원 1명·여러 날짜) 창: 선택한 모든 날짜 공통 퇴근시각 일괄 입력.
  - [신규] 기준 퇴근시간 이후 근무분을 "야간당직 시간"으로 집계 →
    "과별 당직 횟수" 요약에 직원별 시간 병기 (예: 홍길동 5회 · 12시간 30분).
    · 기준 퇴근시간 = 관리자 → 시스템 → 운영 시간의 "야간당직 기준"(기본 18:30,
      운영 종료와 별개). 자정 넘김 퇴근도 정확 계산. 아침당직은 미적용.
  - 검증: 기록 메모 왕복 + 당직 end_time 왕복·초과분 계산 테스트 + ruff.
    DB 마이그레이션 m044(record_entries.memo) + m045(employee_duties.end_time)
    — 컬럼 추가만·보존·멱등.
  - ※ 당초 v1.3.56 으로 준비했으나 정산 확정(잠금)이 v1.3.56 으로 먼저 배포되어
    v1.3.57 로 번호를 재배정.

[이전 변경 (v1.3.56) — 정산 확정(잠금): 지난 정산 금액 자동 고정]
  - [추가] 지난 정산을 자동으로 "확정(잠금)" — 매월 1일에 '2달 전' 달이 확정됨.
    · 예) 7월에는 5월 이하가 확정, 6월·이번 달(7월)은 계속 수정 가능.
      8월 1일이 되면 6월이 자동 확정. 별도 버튼·작업 없이 날짜만으로 자동.
    · 확정된 기간은 그때 계산된 수량·금액이 그대로 고정 — 나중에 집계를 고치거나
      수가·인센티브를 바꿔도 확정된 달의 정산 금액은 더 이상 변하지 않음.
      → 이미 지급한 급여의 근거가 보존됨 (v1.3.55 "항상 최신 재계산" 보완).
    · 단, 한 번도 조회한 적 없는 과거 달은 확정 기간이라도 최초 1회는 집계가 채워짐
      (확정됐다는 이유로 과거 정산이 통째로 비어 보이는 것 방지).
  - [개선] 정산 화면 상단에 확정 상태 안내 표시(앰버색 강조).
  - 검증: 전체 회귀 2,299 passed(환경상 격리 불가 22건 제외) + ruff + node.
    정산 잠금 신규 테스트 3건. DB·마이그레이션 변동 없음.

[이전 변경 (v1.3.55) — 휴무 카운트·과별 보기 + 집계→정산 자동 반영 + 집계 칸 축소]
