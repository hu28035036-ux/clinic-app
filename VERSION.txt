[빌드 정보]
  실행 플랫폼    : Windows x64
  배포 방식       : PyInstaller onedir
  Python 런타임  : 3.12 (내장)
  DB             : SQLite

[주요 변경 (v1.3.0) — AI/RAG 1차 통합]
  ✨ AI 도우미 탭 (업무 매뉴얼 Q&A)
     · 직원이 자연어로 매뉴얼 질문 → 답변 + 출처 + 신뢰도 표시
     · 신규 API:
         POST /api/ai/manual/search  — 키워드 RAG 검색만 (LLM 호출 없음)
         POST /api/ai/manual/ask     — 검색 + LLM 답변
     · 신규 매뉴얼 6종 (knowledge/manuals/):
         예약문자 작성 / 치료사 미배정 시 문자 / 문자나라 오류 대응 /
         백업 / 치료사 휴무 등록 / AI 설정

  ✨ 예약문자 AI 초안 + 점검 (관리자 → AI 설정에서 키 입력 시 활성화)
     · POST /api/ai/sms/validate  — 발송 전 점검 (PII / 번호 / 시각 검증)
     · POST /api/ai/sms/draft     — 환자 컨텍스트 기반 SMS 초안 생성

  🛡️ PII 보호 + 할루시네이션 방어
     · 환자 PII (이름/전화/생년월일/차트번호/메모) 검색·전달 차단
     · 사용자 입력에 PII 패턴이 섞이면 자동 마스킹 후 검색
     · LLM 응답 사후 검증: 매뉴얼에 없는 시간/치료사명 환각 차단
     · AiUsageLog 에 PII 원문 저장 금지 — 해시(prompt_hash/response_hash)만

  📊 AiUsageLog 스키마 확장 (m008)
     · outcome / error_detail / prompt_hash / response_hash
     · pii_filter_hits / hallucination_guard_hits
     · response_used / sms_sent

  ⚙️ AI 설정 테이블 (m007)
     · enabled / provider (openai|anthropic) / model / api_key /
       base_url / max_tokens / temperature / pii_guard_enabled
     · 기본값 enabled=0 — 키 입력 + 활성화 전까지 모든 LLM 호출 차단

  🔒 정책
     · 벡터DB / 임베딩 미사용 — 키워드 RAG 만 (1차)
     · 매뉴얼에 없는 질문은 LLM 호출 없이 "매뉴얼에서 답을 찾지 못했습니다."
     · 기존 예약 / 환자 / 치료사 / 예약문자 / 통계 / 관리자 흐름은 일체 변경 없음
