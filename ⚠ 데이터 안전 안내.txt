  ⚠ 환자 데이터 안전 안내 — 반드시 읽어주세요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


┌────────────────────────────────────────────────┐
│ 1. 환자 데이터가 실제로 저장되는 곳              │
└────────────────────────────────────────────────┘

  C:\Users\<사용자>\AppData\Roaming\도수치료예약\

  ※ 프로그램 폴더 (예: C:\도수치료예약\) 에는
    환자 데이터가 없습니다.
    → 프로그램 폴더를 지우거나 옮겨도 환자 DB 는 안전합니다.


┌────────────────────────────────────────────────┐
│ 2. 절대 삭제하면 안 되는 파일                   │
└────────────────────────────────────────────────┘

  [위 AppData 폴더 안]
    ├── clinic.db           ← 환자 · 예약 · 직원 · 문자 로그
    ├── config.json         ← 관리자 비밀번호 · 설정
    ├── schema_version.txt
    └── backups\            ← 자동 백업본 (사고 시 복구용)

  ⚠ 이 4개 항목 중 하나라도 삭제되면 일부 또는 전체 데이터 손실.


┌────────────────────────────────────────────────┐
│ 3. 프로그램 업데이트 시 동작 방식                │
└────────────────────────────────────────────────┘

  v1.2.2 부터 **증분 마이그레이션 시스템** 으로 동작:

  새 배포본 실행 → schema_migrations 테이블 조회 →
    아직 적용 안 된 변경만 ALTER/CREATE IF NOT EXISTS 로 적용 →
    기존 환자/예약/설정 그대로 유지

  ➜ 업데이트는 언제나 "프로그램 폴더만" 교체.
    AppData 폴더는 자동으로 그대로 유지.


┌────────────────────────────────────────────────┐
│ 4. 권장 백업 주기                               │
└────────────────────────────────────────────────┘

  [자동 백업]
    프로그램이 백그라운드에서 자동으로 backups\ 에 DB 복사.
    관리자 → 시스템 → 자동 백업 카드에서 주기 조정 가능.

  [수동 백업 — 권장 시점]
    ① 프로그램 업데이트 직전
    ② 월 1회 정기
    ③ 대량 import 직전
    ④ 연말 결산 전

    간단 방법:  tools\백업하기.bat 더블클릭
                → 바탕화면에 날짜 붙은 백업 폴더 자동 생성


┌────────────────────────────────────────────────┐
│ 5. 사고 시 복구                                │
└────────────────────────────────────────────────┘

  [경미 — clinic.db 손상만]
    tools\복원하기.bat 더블클릭
    → backups\ 최신 백업으로 자동 복원

  [중대 — AppData 폴더 통째 삭제됨]
    수동 백업본에서 복원:
      1) 프로그램 종료
      2) 백업 폴더를 %APPDATA%\도수치료예약\ 에 붙여넣기
      3) 프로그램 재시작

  [최악 — 백업도 없음]
    프로그램을 종료하지 않고 전원이 꺼지더라도
    SQLite WAL 로그로 대부분 자동 복구됩니다.
    그래도 안 열리면 배포자에게 문의.


┌────────────────────────────────────────────────┐
│ 6. 주의                                        │
└────────────────────────────────────────────────┘

  ✗ clinic.db 를 이메일·카톡·공용 클라우드에 올리지 마세요
    → 환자 개인정보 포함. 개인정보보호법 위반 소지.

  ✗ 여러 PC 에서 같은 clinic.db 를 동시 편집하지 마세요
    → 파일 잠금 충돌로 데이터 꼬임 가능.
    → 여러 PC 사용 시 "메인 서버 1대 + 서브 PC 는 브라우저 접속" 구조 사용.

  ✓ USB / 외장 HDD / NAS 에 AppData 폴더 복사 보관 OK
    → 단 USB 잃어버리지 않게 유의.
